# Festo VTUX Valve Terminal вҖ” CPX-AP Integration Reference

> Pneumatic valve terminal reference for FestoCodesysMCP.
> Successor to CPV, MPA-L/S, and VTUG series.

---

## Product Overview

The Festo VTUX is Festo's flagship valve terminal platform (launched 2025). It combines the functions of three classic valve terminals (CPV, MPA-L/S, VTUG) with advanced communication via the Festo Automation Platform (AP).

## Key Specifications

| Parameter | Value |
|-----------|-------|
| **Max Flow Rate** | Up to 730 l/min (high-flow subbase) |
| **Max Valve Positions** | Up to 128 |
| **Max Solenoid Coils** | Up to 128 |
| **Protection Rating** | IP65 / IP67 |
| **Construction** | High-performance polymer |
| **Vacuum Channels** | Up to 16 per terminal |
| **Process Data** | Input 2 kByte / Output 2 kByte |
| **AP Data Rate** | Up to 200 Mbaud |

## Subbase Options

| Subbase Type | Flow Rate | Valve Positions | Use Case |
|--------------|-----------|-----------------|----------|
| **Compact** | Standard | 1 or 4 per module | Space-saving installations |
| **High-Flow** | Up to 730 l/min | 1 or 4 per module | High flow rate applications |

A single valve model fits both subbases вҖ” mix and match on the same terminal.

## EtherCAT Integration

### ESI File

Download the ESI (EtherCAT Slave Information) file from the Festo Support Portal for the CPX-AP-A-EC-M12 or CPX-AP-I-EC-M12 interface module.

### PDO Mapping

| Object | Type | Description |
|--------|------|-------------|
| 0x1600+ | RxPDO | Receive PDO Mapping вҖ” valve output commands |
| 0x1A00+ | TxPDO | Transmit PDO Mapping вҖ” valve status / feedback |
| 0x1C12 | SM PDO | Sync Manager 2 PDO Assignment (outputs) |
| 0x1C13 | SM PDO | Sync Manager 3 PDO Assignment (inputs) |
| 0x2000+ | Params | Module parameters (0x2000=Module 1, 0x2010=Module 2, etc.) |
| 0x10F3 | Diag | Diagnostic object ("New message available") |

### Module Slot Configuration

Each valve terminal slot and I/O module occupies a slot in the EtherCAT configuration:
- Add modules one by one in CODESYS device tree (do not rely on scan)
- Each module maps to a specific PDO range

## CODESYS Programming

### I/O Addressing

Valve outputs are accessed via `IoConfig_Globals` variables auto-generated by CODESYS when PDO-mapped I/O is linked to the PLC program.

### Valve Control in Structured Text

```iecst
(* Example: Control VTUX valves via CPX-AP-A *)
VAR
    bValve1Fwd    : BOOL;    (* Valve 1 вҖ” extend *)
    bValve1Rev    : BOOL;    (* Valve 1 вҖ” retract *)
    bValve2Single : BOOL;    (* Valve 2 вҖ” single-acting *)
    bVacuumGrip   : BOOL;    (* Vacuum generator ON *)
    bEjectorPulse : BOOL;    (* Ejector pulse *)
END_VAR

(* Map these to CPX-AP-A RxPDO outputs in IoConfig *)
(* Valve outputs are bit-addressed in the output PDO *)
```

### Typical GVL for VTUX

```iecst
VAR_GLOBAL
    (* VTUX Valve Outputs вҖ” mapped to CPX-AP-A RxPDO *)
    bCylGripFwd    AT %QX0.0 : BOOL;   (* Gripper extend *)
    bCylGripRev    AT %QX0.1 : BOOL;   (* Gripper retract *)
    bCylLiftFwd    AT %QX0.2 : BOOL;   (* Lift extend *)
    bCylLiftRev    AT %QX0.3 : BOOL;   (* Lift retract *)
    bCylPushFwd    AT %QX0.4 : BOOL;   (* Push extend *)
    bCylPushRev    AT %QX0.5 : BOOL;   (* Push retract *)
    bVacuumOn      AT %QX0.6 : BOOL;   (* Vacuum generator *)
    bBlowOff       AT %QX0.7 : BOOL;   (* Blow-off / ejector *)

    (* VTUX Valve Inputs вҖ” mapped to CPX-AP-A TxPDO *)
    (* Sensor feedback via CPX-AP I/O modules *)
    bCylGripFwdSns AT %IX0.0 : BOOL;   (* Gripper extended sensor *)
    bCylGripRevSns AT %IX0.1 : BOOL;   (* Gripper retracted sensor *)
END_VAR
```

**Note**: Prefer symbolic addressing via CODESYS device tree mapping over direct `%QX`/`%IX` addresses. The example above shows the concept; actual addresses depend on the CPX-AP-A module configuration.

## Valve Functions

| Function | Description |
|----------|-------------|
| **5/2 double-acting** | Extend/retract cylinder (2 coils) |
| **5/3 center-closed** | Extend/retract/hold (2 coils + spring center) |
| **3/2 single-acting** | On/off (1 coil, spring return) |
| **Vacuum** | Venturi vacuum generator (suction + ejector pulse) |
| **Pressure zones** | Configurable separate pressure supplies |

## Vacuum Integration

- Up to 16 vacuum channels per terminal
- Bidirectional communication PLC вҶ” vacuum generator
- Individual control: vacuum ON + ejector pulse per channel
- Configurable for high vacuum or large suction volume

## Electrical Linkage

| Mode | Communication | Max Coils | Description |
|------|---------------|-----------|-------------|
| **Parallel** | Unidirectional | 32 | Simple direct wiring, multipin connector |
| **Serial** | Bidirectional | 128 | AP communication, advanced diagnostics |

## Diagnostics

- Module-level diagnostics via PDO object 0x10F3
- Coil status feedback (open circuit, short circuit detection)
- Pressure zone monitoring (if equipped)
- AP communication provides predictive maintenance data

## Setup Checklist

1. Configure VTUX terminal in Festo Automation Suite (valve layout, pressure zones)
2. Download ESI file for CPX-AP-A-EC-M12 or CPX-AP-I-EC-M12
3. Install ESI in CODESYS EtherCAT master configuration
4. Add CPX-AP EtherCAT slave in CODESYS device tree
5. Add valve terminal and I/O module slots one by one
6. Map valve outputs to Structured Text variables
7. Test individual valve outputs before running sequences
